# LP品質チェックスキル

design/ のLPをチェックして問題を報告します。

## チェック項目

### 1. タイポグラフィ（375px孤立文字）
```bash
npm run check:typography
```
- 375px幅で1-2文字だけ行末に残る箇所を検出
- 改行位置の調整を提案

### 2. アクセシビリティ
```bash
npm run check:accessibility
```
- alt属性の有無
- lang属性
- 見出し構造
- フォーカス表示
- フォームラベル

### 3. リンクチェック
```bash
npm run check:links
```
- 内部アンカーの確認
- 外部リンクのrel属性
- 画像パスの確認
- CSS/JSファイルの確認

### 4. 全チェック実行
```bash
npm run check:all
```

## 手動確認チェックリスト

### パフォーマンス
- [ ] 画像にwidth/height属性がある
- [ ] ファーストビュー以外の画像にloading="lazy"
- [ ] 大きすぎる画像がない（2MB以上は要圧縮）

### SEO
- [ ] `<title>` が設定されている
- [ ] `<meta name="description">` が設定されている
- [ ] 見出しが論理的な順序（h1→h2→h3）

### コンバージョン
- [ ] CTAボタンにdata-cv属性がある
- [ ] CTAが目立つ位置にある
- [ ] 電話番号リンクにdata-cv="tel"がある

### レスポンシブ
- [ ] 375pxで文字が読みやすい
- [ ] ボタンがタップしやすい（44x44px以上）
- [ ] 横スクロールが発生しない

### 法的要件
- [ ] プライバシーポリシーへのリンク
- [ ] 特定商取引法表記へのリンク（商品販売の場合）

## 問題があった場合

1. 問題箇所を具体的に報告
2. 修正方法を提案
3. ユーザーの確認後に修正を実施

## 完了後

全チェックをパスしたら：
- `/lp-ads` で広告タグ設定
- `/lp-build` で本番ビルド
